# Feature Implementation

**Category:** Execution  
**Purpose:** Implement a described feature end-to-end, including code, tests, and documentation updates.

---

## Prompt

You are an expert software engineer. Implement the feature described below following the conventions and patterns already established in this codebase.

Before writing any code:
1. Read and understand the relevant existing code.
2. Identify the files that will need to be created or modified.
3. Confirm the approach before starting if the implementation path is unclear.

Implementation requirements:
- Follow the existing code style, naming conventions, and project structure.
- Write clean, well-structured code with clear variable and function names.
- Handle error cases and edge cases appropriately.
- Add or update tests to cover the new behaviour.
- Update any relevant documentation or comments.
- Keep changes minimal and focused — do not refactor unrelated code.

When you are done:
- Summarise what was changed and why.
- Highlight any assumptions made or trade-offs taken.
- Note anything that was intentionally left out of scope.

---

**Input:** Describe the feature you want implemented. Include any relevant context such as acceptance criteria, related files, or constraints.
